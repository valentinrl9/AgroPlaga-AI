 AgroPlaga AI — Especificación Técnica Integral y Plan de DesarrolloAutor: Valentín Ruiz LeónEstado: Listo para Desarrollo / PortfolioAño: 20260. Enfoque GeneralEste documento define la estructura analítica, el ecosistema de componentes y el plan estratégico para el desarrollo de AgroPlaga AI. El sistema se conceptualiza como un producto mínimo viable (MVP) de nivel industrial listo para su integración directa en portfolios profesionales, presentaciones ante inversores (pitch) y despliegue real en el sector agrícola inteligente (Smart Farming). La viabilidad y solidez del sistema se garantizan mediante la definición explícita de:Stack Tecnológico Concreto: Selección precisa de frameworks de alta eficiencia y rendimiento. Arquitectura Clara: Desacoplamiento de responsabilidades organizadas por capas lógicas. Estructura de Archivos Normalizada: Andamiaje limpio y escalable tanto para el cliente como para el servidor. Seguridad por Diseño: Cifrado de datos, autenticación robusta y anonimización de la geolocalización. Roadmap Estricto: Ciclo de desarrollo incremental dividido en fases operativas estables. 1. Arquitectura GeneralEl sistema se rige bajo una arquitectura desacoplada cliente-servidor distribuida en las siguientes capas funcionales: A. Aplicación Móvil (Flutter)Capa de Interfaz de Usuario (UI): Vistas modulares y componentes reactivos optimizados para el trabajo en campo. Lógica de Presentación (BLoC/Provider): Control de estados de la interfaz de usuario, gestión de llamadas asíncronas y flujo de datos. Cliente API: Cliente HTTP centralizado para sincronización e interacción de datos persistentes. Motor de Inferencia Local: Módulo encapsulado que ejecuta el archivo compilado .tflite localmente, garantizando un diagnóstico inmediato en zonas sin cobertura de red. B. API Backend (FastAPI)Capa de Autenticación y Seguridad: Controladores de acceso basados en JSON Web Tokens (JWT) y cifrado criptográfico. Módulos de Negocio: Enrutadores específicos dedicados al procesamiento de escaneos, gestión distribuida de focos de infección y generación de analíticas agregadas. C. Capa de Datos (PostgreSQL)Esquema estructurado que almacena de forma íntegra las siguientes entidades: users: Credenciales, datos identificativos y roles del sistema. farms / greenhouses: Delimitación lógica de explotaciones agrícolas asociadas. scans: Historial detallado de capturas con diagnóstico de plagas, índices de confianza y severidad. outbreaks: Focos validados para la alimentación del mapa colaborativo regional. feedback: Datos colaborativos de corrección del usuario para procesos automáticos de reentrenamiento. 3. Estrategia de Seguridad y PrivacidadControl de Accesos Obligatorio: Flujo estricto mediante JSON Web Tokens (JWT) compuestos por tokens de acceso de corta duración y tokens de refresco seguros. Las contraseñas se almacenan procesadas irreversiblemente mediante el algoritmo de hash bcrypt. Autorización Basada en Roles (RBAC): Segmentación rígida de endpoints según jerarquía: farmer (acceso privado a explotación), tech (vista agregada de cooperativa) y admin (control total de la plataforma). Anonimización de Posición (Privacidad Crítica): Para proteger la competitividad comercial e intimidad de las explotaciones, el sistema nunca comparte coordenadas GPS exactas en el mapa público. Las coordenadas de captura se mapean directamente hacia distritos o zonas agrarias predefinidas (ej. &quot;Santa María del Águila&quot;, &quot;Balerma&quot;), eliminando cualquier trazabilidad específica. Robustez Operativa: Validación estricta en tiempo de ejecución en la pasarela de la API mediante esquemas Pydantic, mitigando inyecciones de código. Control de tasa de peticiones (Rate Limiting) activo para mitigar ataques de fuerza bruta en los endpoints de autenticación. 5.
